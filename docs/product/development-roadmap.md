# AI Life OS — Development Roadmap

## Canonical status

Это единственный канонический roadmap продуктовых фаз. Нумерация фаз в других документах не используется для отчетности; технические последовательности там считаются только milestones.

## Purpose

Этот документ превращает Vision в последовательность небольших, независимых шагов. Каждый этап должен завершаться работающим результатом. Нельзя переходить к следующему этапу, пока предыдущий не приносит ценность.

## Phase 0 — Foundation

1. Создать монорепозиторий.

2. Настроить CI, линтеры, форматтеры, strict TypeScript.

3. Зафиксировать PostgreSQL как будущую основную БД без проектирования продуктовой схемы.

4. Поддерживать четыре state-файла и проверку контекста без расширения процесса.

> Аутентификация не входит в Phase 0. Она добавляется отдельной задачей только после появления удаленного хранения или нескольких пользователей.

## Phase 1 — Memory First

1. Реализовать Memory Engine.

2. Единая строка ввода.

3. Сохранение любого ввода как Memory.

4. История событий.

5. Поиск по памяти.

## Phase 2 — AI Understanding

1. Классификация записей.

2. Извлечение целей, задач, расходов, идей.

3. Связывание сущностей.

4. Пояснение выводов AI.

## Phase 3 — HUD

1. Главный экран.

2. Life Stats.

3. Баланс.

4. Главная цель.

5. Три приоритетных действия.

6. Daily Insight.

## Phase 4 — Finance

1. Доходы и расходы.

2. Финансовые цели.

3. Прогноз накоплений.

4. Анализ долгов.

5. AI-рекомендации.

## Phase 5 — Companion

1. Советник.

2. Хранитель.

3. Стратег.

4. Хранитель памяти.

5. Естественный диалог.

## Phase 6 — Attention Guardian

1. Детектор перегрузки.

2. Фокус-сессии.

3. Информационный фильтр.

4. Еженедельные отчеты.

## Phase 7 — Integrations

1. Календарь.

2. Почта.

3. Заметки.

4. Фитнес.

5. Банки (при возможности).

## Phase 8 — Intelligence

1. Долгосрочные закономерности.

2. Прогноз целей.

3. Персональные стратегии.

4. Глубокие инсайты.

## Phase 9 — Public Beta

1. Закрытое тестирование.

2. Исправление UX.

3. Оптимизация стоимости AI.

4. Подготовка публичного релиза.

## User-visible milestones

M0: Web и API запускаются, проверки проходят.
M1: Пользователь вводит мысль, исходный текст надежно сохраняется и виден в истории.
M2: AI понимает введенные данные.
M3: Пользователь ежедневно открывает HUD.
M4: Финансы и цели реально помогают принимать решения.
M5: Компаньон становится причиной возвращаться в приложение.
M6: Продуктом можно пользоваться каждый день без сторонних органайзеров.

## Definition of Success

После каждого этапа задать вопросы:
- Стал ли продукт полезнее?
- Уменьшилась ли когнитивная нагрузка?
- Соответствует ли изменение Manifesto?
- Можно ли показать результат пользователю уже сейчас?
