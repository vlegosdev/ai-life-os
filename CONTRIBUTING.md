# Contributing

All contributors, human or AI, must follow `AGENTS.md` and the canonical documents in `docs/`.

## Workflow

1. Read current state and next task.
2. Create or select a narrowly scoped task.
3. Explain the plan before implementation.
4. Implement and test.
5. Self-review against product and architecture rules.
6. Update project state and handover.
7. Submit a pull request using the repository template.

Architecture changes require an ADR. Product-direction changes require explicit Product Owner approval.
